# Stroke Prediction with Machine Learning: A Reality Check Using NHANES Clinical Data

## Abstract

**Background.** Machine learning models for stroke prediction routinely report accuracy exceeding 95%, yet these results are derived predominantly from a single Kaggle dataset (5,110 records, 10 self-reported features) without external validation, clinical baseline comparison, fairness evaluation, or calibration analysis.

**Methods.** We trained logistic regression, Random Forest, and XGBoost on six cycles of the National Health and Nutrition Examination Survey (NHANES, 2007–2018; N = 34,719; 1,398 stroke cases; 22 clinically measured features including laboratory values and examiner-measured blood pressure). We used temporal validation (train 2007–2014, test 2015–2018), three class imbalance strategies (none, cost-sensitive, SMOTE), PR-AUC as the primary metric, bootstrap 95% confidence intervals, DeLong tests, a simplified Framingham risk score baseline, and fairness analysis by sex and race/ethnicity. We followed the TRIPOD reporting guidelines.

**Results.** Without class balancing, all models achieved 95.6–95.7% accuracy but detected 0–9 of 482 stroke cases (recall 0.0–1.9%), confirming the "accuracy trap." The best model — logistic regression with cost-sensitive weighting on the full clinical feature set — achieved PR-AUC = 0.162 (95% CI: 0.142–0.188) and ROC-AUC = 0.823 (95% CI: 0.807–0.839). No model pair differed significantly on ROC-AUC (DeLong p > 0.74). Adding clinical features to the Kaggle-equivalent set improved PR-AUC by 12.9% (0.143 to 0.162). The best ML model marginally outperformed a Framingham-style clinical score (PR-AUC: 0.162 vs. 0.144) but with worse calibration (Brier: 0.151 vs. 0.092). Recall ranged from 48.6% (Hispanic) to 76.1% (Non-Hispanic White), revealing a 27-percentage-point fairness gap unreported by any prior study.

**Conclusions.** The high accuracy reported in the stroke prediction literature reflects class imbalance artifacts, not genuine predictive performance. On real clinical data with proper methodology, ML models offer modest discrimination that barely exceeds simple clinical tools, with significant calibration deficiencies and demographic disparities. Advancing this field requires richer clinical data, honest evaluation metrics, clinical baselines, and equity-aware analysis — not more complex models trained on the same limited dataset.
